Note:

The request URL, https://chimepalden.github.io/profile/home returns 404 error while displaying home page.

Possible cause:

A static server routinely returns index.html when it receives a request for "https://www.mysite.com/"; However, it returns 404 error for deep links unless it is configured to return to index.html instead.

The application uses the Angular router and home is component inside the app. So, the requested URL becomes deep link. The URL is handled by browser itself, outside the running application, the browser makes a direct request to the server, bypassing the router.

Solution:

Github Pages server, github.com is a static server and it cannot be configured directly. 404 page is added by creating 404.html file with the copy of index.html content into it. We will get 404 response but the browser will process the page and it will load the app properly.

Detail: https://angular.io/guide/deployment
